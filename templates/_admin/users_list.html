Manage Users and User-Roles

Below are two tables. The first lists all currently registered users in the system.
The second table lists all the defined user-roles. A Role is basically a permissions-set that defines what a user can and can't access/use.
The system requires that the anonymous user remains in the database, as well as at least two roles that define admin privilages and Guest privilages.
